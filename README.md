# 👁️ OpticFSM

**Декларативный фреймворк для неинвазивного Black-box тестирования UI на базе компьютерного зрения и конечных автоматов.**

OpticFSM решает проблему автоматизации и тестирования сложных пользовательских интерфейсов (WebGL, HTML5 Canvas, RDP-сессии, браузерные игры, 3D-конфигураторы), где традиционные инструменты (Selenium, Playwright) бессильны из-за отсутствия DOM-дерева. Фреймворк работает исключительно с пикселями на экране, имитируя поведение реального пользователя.

## ✨ Ключевые особенности

* 🏗 **Data-Driven архитектура (DSL):** Логика тестирования полностью отвязана от кода. Сценарии описываются в формате JSON, что позволяет добавлять новые проекты без знания Python.
* 🧠 **Умное распознавание (OpenCV + MSS):** Поиск визуальных микро-якорей с использованием `Template Matching` (в оттенках серого). Молниеносный захват области интереса без системных рамок Windows.
* 🤖 **Стохастическая гуманизация (Human-like delays):** Модуль задержек поддерживает нормальное (Гауссово) распределение кликов для обхода эвристических антифрод-систем.
* 🛡 **Отказоустойчивость (Watchdog & Limits):** Защита от бесконечного зависания, лимиты сессий и экстренная остановка при появлении визуальных триггеров (например, капчи).
* ✅ **Строгая валидация (Pydantic):** 100% защита от ошибок в конфигурационных файлах благодаря строгой типизации.

## 🚀 Установка и запуск

### 1. Подготовка окружения

Рекомендуется использовать виртуальное окружение (venv). Для установки всех зависимостей выполните команду:

```bash
python -m pip install pydantic opencv-python mss pygetwindow pyautogui numpy
```

### 2. Структура проекта
Рекомендуемая структура директорий для работы с фреймворком:
```text
OpticFSM/
├── src/                  # Исходный код ядра
│   ├── config.py         # Pydantic модели и парсер DSL
│   ├── core.py           # Логика графа конечного автомата
│   ├── vision.py         # Адаптер компьютерного зрения
│   └── main.py           # Точка входа
├── examples/             # Примеры сценариев (Тестовые стенды)
│   └── erp_test_stand/
│       ├── config.json   # Конфигурация сценария
│       └── templates/    # Нарезанные скриншоты (якоря и цели)
└── README.md
```

### 3. Быстрый старт
Для запуска бота передайте ему путь к конфигурационному файлу (по умолчанию main.py ищет config.json в текущей директории):

```bash
python src/main.py
```

## 📖 Справочник конфигурации (DSL)
Вся магия OpticFSM скрыта в файле config.json. Файл состоит из двух основных блоков: глобальных настроек и описания состояний.

### 1. Глобальные настройки (engine_settings)
Этот блок управляет поведением окна ОС и модулем имитации человека.

```json
"engine_settings": {
  "project_name": "My_First_Test",
  "target_window_title": "Имя окна в Windows",
  "confidence_threshold": 0.85, 
  "global_timeout_sec": 60,
  "delay": {
    "strategy": "gaussian", 
    "mean": 0.9,
    "std_dev": 0.2
  },
  "session_limits": {
    "max_iterations": 50,
    "iteration_trigger_state": "success_end",
    "stop_anchors": ["templates/error_captcha.png"]
  }
}
```

- confidence_threshold: Порог уверенности OpenCV (от 0.0 до 1.0). Рекомендуется 0.85-0.92.

- delay.strategy: Поддерживает fixed (жесткая задержка), uniform (равномерный разброс) и gaussian (распределение Гаусса для имитации человека).

- session_limits: Опциональный блок для цикличных автотестов.


### 2. Конечный автомат (states)
Граф состоит из набора состояний. Каждое состояние должно быть подтверждено Якорями (Anchors), после чего фреймворк выполнит Переходы (Transitions).

```json
"states": {
  "dashboard_menu": {
    "is_terminal": false,
    "anchors": [
      "templates/logo.png",
      "templates/settings_icon.png"
    ],
    "transitions": [
      {
        "target_template": "templates/btn_start.png",
        "action": "click",
        "next_state": "loading_screen",
        "delay_msec": null
      }
    ]
  }
}
```

- anchors (Список путей к изображениям): Логическое "И". Стейт считается активным, только если ВСЕ якоря найдены на экране.

- transitions: Массив действий. Выполняются строго по порядку (Ленивые вычисления). Как только срабатывает первый переход, автомат меняет состояние.

- action: Доступные действия — click, double_click, wait. Для действия wait обязательно наличие параметра delay_msec (пауза в миллисекундах).

### 🗂 Спецификация конфигурационного файла (config.json)

Ниже приведено подробное описание каждого параметра, его тип и обязательность.

#### 1. Корневой объект
| Поле | Тип | Статус | Описание |
| :--- | :--- | :--- | :--- |
| `engine_settings` | `Object` | **Обязательно** | Глобальные настройки движка, поиска изображений и поведения ОС. |
| `start_state` | `String` | **Обязательно** | Имя состояния, с которого конечный автомат начнет свою работу при запуске. |
| `states` | `Object` | **Обязательно** | Словарь всех возможных состояний. Ключ — имя состояния, значение — объект конфигурации. |

#### 2. Глобальные настройки (`engine_settings`)
| Поле | Тип | Статус | Описание |
| :--- | :--- | :--- | :--- |
| `project_name` | `String` | **Обязательно** | Название проекта. Используется исключительно для красоты в логах. |
| `target_window_title` | `String` | **Обязательно** | Точный заголовок окна в ОС Windows. Движок сам найдет окно по этому тексту. |
| `confidence_threshold` | `Float` | Опционально | Порог уверенности OpenCV (от 0.0 до 1.0). По умолчанию: 0.85. |
| `global_timeout_sec` | `Int` | **Обязательно** | Максимальное время (в секундах), которое автомат может провести в одном состоянии. |
| `delay` | `Object` | **Обязательно** | Настройки модуля стохастических задержек для кликов. |
| `session_limits` | `Object` | Опционально | Лимиты работы сессии. Если блок удалить, скрипт будет работать бесконечно. |

#### 3. Настройки задержек (`delay`)
| Поле | Тип | Статус | Описание |
| :--- | :--- | :--- | :--- |
| `strategy` | `String` | Опционально | Тип имитации человека: `fixed`, `uniform` или `gaussian`. По умолчанию: `gaussian`. |
| `mean` | `Float` | Опционально | Базовая задержка в секундах (мат. ожидание). По умолчанию: 0.8. |
| `std_dev` | `Float` | Опционально | Стандартное отклонение для распределения Гаусса. По умолчанию: 0.2. |
| `min_val` | `Float` | Опционально | Минимальное время задержки (для uniform). |
| `max_val` | `Float` | Опционально | Максимальное время задержки (для uniform). |

#### 4. Лимиты сессии (`session_limits`)
| Поле | Тип | Статус | Описание |
| :--- | :--- | :--- | :--- |
| `max_iterations` | `Int` | Опционально | Сколько циклов должен пройти бот до завершения работы. |
| `iteration_trigger_state`| `String` | Опционально | Имя состояния, при входе в которое счетчик итераций увеличивается на 1. |
| `max_runtime_sec` | `Int` | Опционально | Абсолютный лимит времени работы скрипта (в секундах). |
| `stop_anchors` | `Array[Str]`| Опционально | Список путей к изображениям (капча, ошибки). При их обнаружении скрипт останавливается. |

#### 5. Состояния (`states -> [state_name]`)
| Поле | Тип | Статус | Описание |
| :--- | :--- | :--- | :--- |
| `is_terminal` | `Bool` | Опционально | Если `true`, бот штатно завершает работу при попадании в этот стейт. По умолчанию: `false`. |
| `anchors` | `Array[Str]`| **Обязательно** | Список путей к картинкам-якорям (Логическое И). |
| `transitions` | `Array[Obj]`| **Обязательно** | Список переходов, которые автомат будет пытаться выполнить в этом состоянии. |

#### 6. Переходы (`transitions`)
| Поле | Тип | Статус | Описание |
| :--- | :--- | :--- | :--- |
| `action` | `String` | **Обязательно** | Действие: `click`, `double_click` или `wait`. |
| `target_template` | `String` | Условно* | Путь к картинке-цели. Обязательно для действий `click` и `double_click`. |
| `next_state` | `String` | **Обязательно** | Имя состояния, в которое перейдет автомат после успешного выполнения действия. |
| `delay_msec` | `Int` | Условно* | Время паузы в миллисекундах. Обязательно для действия `wait`. |
| `retries` | `Int` | Опционально | Количество попыток кликнуть по цели. По умолчанию: 1. |

## ✂️ Правила нарезки шаблонов (Best Practices)
Качество работы фреймворка напрямую зависит от того, как подготовлены визуальные шаблоны (изображения в папке templates/).

1. Микро-якоря лучше полноэкранных шаблонов: Никогда не вырезайте весь экран. Ищите маленькие, статичные и уникальные иконки (логотипы, шестеренки настроек) в углах экрана. Это радикально повышает производительность OpenCV.

2. Избегайте динамического фона: Если кнопка полупрозрачная и сквозь нее видно анимацию или разный фон — вырезайте только центральную часть кнопки (текст или иконку), оставляя фон за бортом.

3. Осторожно со сжатием: Сохраняйте шаблоны в формате .png без потерь качества (lossless). Использование .jpg может создать визуальные артефакты (шум), которые снизят процент совпадения (confidence) ниже установленного порога.
